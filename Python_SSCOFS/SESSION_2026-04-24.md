## Open questions / known issues

1. **Inconsistent penalty sweep behavior.** Sometimes a sweep across `tack_penalty_s` shows clear path differences (e.g., short upwind: pen 6→7 tacks, 60→5, 600→1). Sometimes it shows zero variation (long round-trip pen 0/15/30/60/90/120 all giving 5 tacks / 203.7 min). I labeled the no-variation cases "script artifact" or "5-tack solution geometrically dominant", but I never actually proved either. The pattern may be related to:
   - In-place YAML sed in a bash loop possibly racing the conda subprocess startup with the prior run's npz flush
   - Numba JIT cache reuse interacting badly with rapid successive runs
   - Or something else entirely

   **Test that resolves it:** running each penalty value as a separate YAML file (or fully synchronous individual runs) consistently shows clear differentiation. Until the in-place-sed-loop pattern is understood or avoided, sweeps should use separate YAMLs.

2. **Cumulative-turn loophole in tack penalty.** A* can dodge the per-segment penalty by splitting one ~90° corner turn across 3 raw segments of ~30° each, none individually exceeding the 50° threshold. The pen=600 one-way upwind shows 0 "tacks > 50°" by this metric while having visible 90°-ish corner turns on the plot. Doesn't bite at typical pen=60-90 settings but is a real flaw. Two reasonable fixes:
   - Cumulative-turn penalty (sum heading change over a sliding window)
   - Smoothed-waypoint penalty pass (compute turns between *kept* waypoints post-smoothing)

3. **Code drift across A* implementations.** SectorRouter has at least 2 paths (Numba JIT + Python fallback) plus MeshRouter exists. Risk of fixes diverging across them. User flagged this as a concern. Deferred to post-race cleanup.

4. **Pre-existing test failure** in `test_sail_routing.py::TestPathSmoothing::test_line_of_sight_blocked` — legacy `Router._line_of_sight` test, untouched by this work, was already failing before any changes today. 80 of 81 tests pass.

5. **Wind source choice.** Race YAML uses manual `wind.schedule` (Paul's forecast). ECMWF live forecast is an alternative via `source: open_meteo_ecmwf` — not exercised today. Worth comparing on race morning.
